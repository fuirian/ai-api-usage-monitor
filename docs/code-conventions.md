# 코드 컨벤션 (네이밍 · 스타일)

팀 전체가 따르는 **정본** 문서이다. 상세 규칙은 본 문서를 기준으로 하며, AI 보조 도구용 요약은 `.cursor/rules/code-conventions-pointer.mdc`에 둔다.

---

## 1. 네이밍 규칙

### 1.1 공통

- 대문자와 소문자를 엄격히 구분한다.
- 축약보다 **의미가 드러나도록** 풀어 쓴다.
- 이름은 **숫자로 시작할 수 없다.**

### 1.2 변수·메서드 (Java)

- **lowerCamelCase**를 사용한다.
- **변수명**은 명사로 쓴다.  
  예: `name`, `userId`, `usageCount`
- **메서드명**은 동사 또는 동사구(동사 + 목적어)로 쓴다.  
  예: `getName()`, `findPassword()`, `getUserId()`, `getUsage()`
- **`boolean` 변수**는 다른 변수와 동일하게 **lowerCamelCase**로 쓰고, 의미상 필요하면 `is` 접두어를 쓴다.  
  예: `isHuman`, `isActive`

### 1.3 상수명 표기

- 상수(의미 있는 불변 값)는 관례상 **`UPPER_SNAKE_CASE`**(대문자 + 언더스코어)로 표기한다.  
  예: `AI_API_KEY`, `MAX_RETRY_COUNT`
- 위는 **표기 관례**이며, `enum` 사용 여부·필드 형태 등 구현 방식은 개발자 판단에 맡긴다.

### 1.4 타입·패키지

- **클래스·인터페이스·enum 이름**은 **UpperCamelCase(PascalCase)** 로 쓰고, 명사로 표기한다.  
  예: `UserService`, `UsageRecord`
- **패키지명**은 세그먼트마다 **소문자만** 사용하고, 단어 구분은 **점(`.`)** 으로만 한다. (중간에 CamelCase를 넣지 않는다.)  
  예: `com.example.userservice`

### 1.5 DB (테이블·컬럼)

- **소문자**와 **`snake_case`** 를 사용한다.  
  예: `usage_log`, `user_id`, `created_at`

---

## 2. 코딩 스타일 컨벤션

### 2.1 들여쓰기

- **스페이스 4칸**으로 들여쓴다.
- **탭(Tab) 문자**로 들여쓰기 하지 않는다.

### 2.2 주석

- **코드는 무엇을 하는지**, **주석은 왜 그렇게 하는지(의도·제약)** 를 드러낸다.
- 언어 문법을 그대로 설명하는 주석은, 정말 필요한 경우가 아니면 달지 않는다.
